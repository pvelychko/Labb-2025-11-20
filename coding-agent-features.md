# Coding Agent Features Comparison


| Capability                                                         | Claude Code CLI                                                                                                                                                                                                                                                                                     | Gemini CLI                                                                                                                                                                                                                                                                                               | Codex CLI                                                                                                                                                                                                                                                                                                                                        | GitHub Copilot CLI                                                                                                                                                                                                                                                                                        |
| ------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Parameterized custom slash commands / prompts**                  | **Yes.** Custom commands are Markdown files in `.claude/commands` or `~/.claude/commands`. Arguments via `$ARGUMENTS`, `$1`, `$2`, etc.; frontmatter supports `description`, `argument-hint`, `model`, `allowed-tools`, etc. ([code.claude.com][1])                                                 | **Yes.** Custom commands in TOML under `~/.gemini/commands` and `<project>/.gemini/commands`. Parameters via `{{args}}` (raw or shell-escaped), or by appending the raw command when `{{args}}` is absent. ([Gemini CLI][2])                                                                             | **Yes.** Custom prompts become slash commands under `/prompts:<name>` using Markdown files (e.g. `~/.codex/prompts/draftpr.md`) with frontmatter for `description`, `argument-hint`; arguments are passed through and surfaced in the prompt. ([OpenAI Developers][3])                                                                           | **No first-class custom slash commands.** CLI exposes a fixed set of subcommands (e.g. `github-copilot-cli chat`, `… explain`, `… tests`) and uses **custom instructions** instead of user-defined slash commands. You can’t define new CLI commands the way Claude/Gemini/Codex do. ([GitHub Docs][4])   |
| **File “@-mentions” / inline file references in prompts/commands** | **Yes.** In **slash commands** you can include files with `@path/to/file` (or multiple `@…`) which inlines file contents into the command context. ([code.claude.com][1]) Also **memory** (CLAUDE.md) can import other files via `@path` syntax, including home-dir imports. ([code.claude.com][5]) | **Yes.** In **custom commands**, `@{path}` injects file contents or directory listings (respecting `.gitignore` / `.geminiignore`), including multimodal inputs. ([Gemini CLI][2]) In interactive mode you can also bring files into context then trigger commands.                                      | **Partial.** There is a `/mention` built-in slash command to attach a file or folder to the conversation (`/mention path/to/file.ts`). ([OpenAI Developers][3]) Codex docs don’t currently describe an inline `@file` macro like Claude/Gemini; instead you highlight files via `/mention` and via AGENTS.md discovery. ([OpenAI Developers][6]) | **No dedicated `@file` macro in docs.** Copilot CLI uses repository context plus **repository custom instructions** (`.github/copilot-instructions/copilot-instructions.md`) and personal instructions, but file references are just text in the prompt, not a structured `@` feature. ([GitHub Docs][7]) |
| **Persistent instruction files (project / user)**                  | `CLAUDE.md` hierarchy: enterprise, project, user; CLI loads and can **import** additional files with `@path`. `/memory` opens and edits these; `#` shortcut adds new memories. ([code.claude.com][5])                                                                                               | Uses a **Memory tool** + “Memory Import Processor” and settings, plus `.gemini` folder + extensions; not a single AGENTS.md-style file but similar concept via tools and config. ([Gemini CLI][8])                                                                                                       | **AGENTS.md stack.** Global `~/.codex/AGENTS.md` + per-repo and nested `AGENTS.md` / `AGENTS.override.md` (and configurable fallback names). Merged in precedence order on each run. ([OpenAI Developers][6])                                                                                                                                    | **Personal & repo instructions.** Personal instructions stored in your GitHub account; repo-specific instructions via `copilot-instructions.md` under `.github/copilot-instructions/`, including a variant for Copilot CLI. ([GitHub Docs][7])                                                            |
| **Checkpointing / rewind conversation & code**                     | **Rich checkpointing.** Automatic checkpoints before each edit; `/rewind` or `Esc Esc` lets you rewind **conversation**, **code**, or both. Checkpoints persist across sessions but only track edits made via Claude’s tools (not arbitrary bash / external edits). ([code.claude.com][9])          | **Rich checkpointing.** When tools modify files, Gemini creates a **shadow git snapshot + conversation + tool call**. Enable via `settings.json` and use `/restore` to list and restore checkpoints, which restores files **and** conversation and re-proposes the original tool call. ([Gemini CLI][8]) | **Undo / limited checkpointing.** `/undo` rolls back Codex’s most recent turn, including file changes, plus you still have Git history. There’s no separate multi-checkpoint browser like Claude/Gemini; instead you rely on `/undo` + git and session logs. ([OpenAI Developers][3])                                                            | **No dedicated checkpointing feature.** Copilot CLI suggestions are meant to be applied via Git (and in some flows Copilot can run the commands you choose). Revert is via your normal VCS, not a built-in checkpoint UI. ([GitHub Docs][4])                                                              |
| **Resuming previous conversations / sessions**                     | **Yes.** Claude Code stores sessions and checkpoints; checkpoints “persist across sessions” and are accessible when you resume. ([code.claude.com][9]) Docs also describe “resumable subagents” with `agentId` that can be resumed later with full context. ([code.claude.com][10])                 | **Partial.** Checkpointing restores both project files and **conversation history** when you `/restore <checkpoint>`. ([Gemini CLI][8]) There isn’t (in the docs) a named `gemini resume` command analogous to `codex resume`.                                                                           | **Yes (strong).** `codex resume`, `codex resume --last`, or `codex resume <SESSION_ID>` re-open interactive sessions with full transcript, plan history, approvals, etc.; `codex exec resume` resumes non-interactive runs as well. ([OpenAI Developers][11])                                                                                    | **Not documented as a feature.** Each `github-copilot-cli` invocation starts a fresh interactive session. Docs don’t describe a way to resume prior CLI chats from disk. ([GitHub Docs][4])                                                                                                               |
